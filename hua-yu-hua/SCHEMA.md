# 华与华方法 Wiki Schema

> 基于 Karpathy LLM Wiki 方法论，从 `raw-articles/hua-yu-hua/methods/` 和 `raw-articles/hua-yu-hua/speeches/` 的源文件编译而来。

## 域（Domain）

华与华品牌营销方法论体系。涵盖企业战略、品牌战略、传播三大板块，15个核心概念。

**来源**：华杉公众号文章、公开演讲、内部培训实录。
**分类**：🔴 华与华外脑，仅供内部学习参考，不可直接用于客户提案。

## 约定

- 文件名：中文，使用连字符 `-`（与源文件保持一致）
- 每个概念页使用 YAML frontmatter
- 使用 ```概念页名`` 格式建立交叉引用（匹配实际文件名，不含 `.md`）
- 更新页面时，更新 `updated` 日期
- 新增概念页必须加入 `index.md` 并记录到 `log.md`
- 概念页综合 3+ 源文件时，标注出处 `^[methods/源文件名]`

## Frontmatter

```yaml
---
title: 概念名称
type: concept
source_origin: hua-yu-hua
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [标签1, 标签2]
sources: [methods/源文件1.md, speeches/源文件2.md]
confidence: high | medium | low
---
```

## 概念页规范

每个概念页包含：

1. **一句话定义**：概念的核心含义
2. **核心论述**：从多源综合，保留华杉原话和推理逻辑
3. **关键引用**：适合内部参考的金句
4. **与其他概念的关系**：``交叉引用``
5. **源出处**：末尾列出参考的所有源文件

## 页面阈值

- **创建概念页**：对应模型图中的一个核心概念，且至少 3 篇源文件涉及
- **不创建独立页**：仅在单篇源文件中出现、非核心概念层级的概念，合并在父概念页中
- **拆分页面**：超过 300 行时拆分为子概念页

## 引用指向

| 目标 | 链接路径 |
|------|---------|
| hua-yu-hua 概念 | `概念名` |
| hua-yu-hua 源文件 | `[源文件名](../../raw-articles/hua-yu-hua/methods/源文件.md)` |
| wit-wiki 方法论（主 wiki） | `[链接](../../methodology/页面名.md)` |

## 更新策略

- 新源文件与现有概念页冲突时：标注双方观点和日期，在 frontmatter 中加 `contested: true`
- 不静默覆盖
