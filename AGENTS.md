> 本文件指导 LLM Agent 如何操作 wit-wiki 知识库。
> 基于 Karpathy LLM Wiki 方法论，结合智讯业务特性定制。

## 1. Wiki 架构

```
wit-wiki/
├── raw-articles/          # 原始资料层（只读，已编译）
│   └── [2582个原始文件]
├── cases/                 # Wiki层：智讯案例（150个）
├── methodology/           # Wiki层：智讯方法论（19篇）
├── industries/            # Wiki层：行业洞察
├── hua-yu-hua/            # Wiki层：华与华外脑（内部学习）
├── competitors/           # Wiki层：竞品外脑（内部学习）
├── clients/               # Wiki层：客户画像
├── synthesis/             # Wiki层：综合分析
├── index.md               # 主索引
└── AGENTS.md              # 本文件（Schema层）
```

### 三层架构说明

| 层级 | 目录 | 说明 |
|------|------|------|
| **原始资料层** | `raw-articles/` | 已编译完成，LLM 只读不修改 |
| **Wiki层** | `cases/` `methodology/` 等 | LLM 可读写，持续维护 |
| **Schema层** | `AGENTS.md` `SCHEMA.md` | 操作规范，LLM 必须遵循 |

## 2. 来源标识体系（严格区分）

| 来源 | 标识 | 颜色 | 用途 | 限制 |
|------|------|------|------|------|
| 智讯自有 | `wit` | 🔵 蓝色 | 证明能力、复用经验 | 可用于客户提案 |
| 华与华外脑 | `hua-yu-hua` | 🔴 红色 | 理论框架参考 | **仅供内部学习** |
| 竞品外脑 | `competitor` | ⚪ 灰色 | 竞争情报 | **仅供内部学习** |

**红线规则**：
- 🔴 华与华和 ⚪ 竞品内容 **绝对不可** 直接用于客户提案
- 引用时必须标注来源 `[R: 文件名]`
- 生成客户内容时，优先使用 🔵 智讯自有案例

## 3. 核心操作流程

### 3.1 灌入（Ingest）

将新资料整合进 Wiki。

**触发条件**：
- 新案例需要归档
- 新方法论需要沉淀
- 行业洞察需要更新

**操作步骤**：

1. **读取原始资料**
   - 如果是新案例，先放入 `raw-articles/`
   - 读取并理解核心内容

2. **创建/更新 Wiki 页面**
   - 根据内容类型选择目录：`cases/` 或 `methodology/` 等
   - 使用正确的 Frontmatter 模板（见 SCHEMA.md）
   - 添加来源标识块（🔵🔴⚪）

3. **更新索引**
   - 在 `index.md` 中添加新页面链接
   - 确保没有"孤儿页面"

4. **建立交叉引用**

#### 3.1.1 交叉引用规则

**重要**：`[[页面名]]` 必须匹配**实际文件名**（不含 `.md` 扩展名）。

**正确示例**：
- `[[反碎片品牌方法论]]` → 文件名是 `反碎片品牌方法论.md`
- `[[城市公关的情感战-如何做到千城千面]]` → 文件名使用连字符 `-`

**错误示例**：
- `[[反碎片]]` → 文件名是 `反碎片品牌方法论.md`，不是 `反碎片.md`
- `[[城市公关的情感战：如何做到千城千面]]` → 文件名使用连字符 `-`，不是冒号 `：`

**查找正确文件名**：
1. 在 Obsidian 中打开目标页面
2. 查看页面标题或文件名
3. 使用精确的文件名作为链接目标

**华与华概念引用**：
- WIT 案例可以引用华与华概念（如 `[[超级符号理论]]`）
- 华与华概念页面在 `methodology/` 目录下有别名页
- 引用时使用别名页的文件名

   - 使用 `[[页面名]]` 格式链接相关页面
   - 案例页面链接到相关方法论
   - 方法论页面聚合引用它的案例

5. **记录日志**
   - 更新 `log.md` 记录本次灌入

### 3.2 提问（Query）

向 Wiki 提问，获取带引用的回答。

**操作步骤**：

1. **搜索相关页面**
   - 使用关键词搜索 `cases/` `methodology/` 等目录
   - 优先读取 `index.md` 定位相关主题

2. **读取并综合**
   - 读取相关页面的完整内容
   - 提取关键信息
   - 注意来源标识（🔵🔴⚪）

3. **生成回答**
   - 回答必须包含引用 `[R: 文件名]`
   - 如果涉及 🔴⚪ 来源，明确标注"仅供内部参考"
   - 可生成多种格式：Markdown、对比表格、PPT大纲等

4. **归档优质回答**
   - 有价值的回答可保存到 `synthesis/` 目录
   - 作为新的知识资产沉淀

### 3.3 巡检（Audit）

定期检查 Wiki 质量，发现矛盾、补缺失。

**巡检清单**：

- [ ] **来源一致性**：所有页面都有正确的 `source_origin` 和来源标识块
- [ ] **孤儿页面检查**：所有页面都被 `index.md` 引用
- [ ] **交叉引用完整性**：案例是否链接到相关方法论？
- [ ] **矛盾发现**：同一概念在不同页面是否有冲突描述？
- [ ] **缺失发现**：热门客户/行业是否有足够案例覆盖？
- [ ] **重复检查**：同一概念是否有多个页面（应合并）

**巡检频率**：每月一次

## 4. 页面操作规范

### 4.1 创建新页面

**必须包含**：
1. Frontmatter（按 SCHEMA.md 规范）
2. 来源标识块（页面开头）
3. 正文内容
4. 至少一个数据点（案例页）
5. 交叉引用链接

**案例页模板**：
```markdown
---
type: case
source_origin: wit
title: "案例名称"
date: 2024
industry: 房地产
client: "客户名"
methodology: ["反碎片", "城市品牌"]
keywords: [关键词]
decision_logic: "一句话决策逻辑"
source: "原始文章目录名"
---

> 🔵 **智讯自有** · 案例

## 背景

## 策略

## 执行

## 效果

## 方法论链接
- [[相关方法论]]
```

### 4.2 更新现有页面

- 保持来源标识不变
- 新增内容标注更新时间
- 如发现矛盾，标记并报告
- 更新交叉引用

### 4.3 删除页面

**禁止随意删除**。如需删除：
1. 检查是否有其他页面引用
2. 在 `index.md` 中移除链接
3. 在 `log.md` 记录删除原因

## 5. 内容生成规范

### 5.1 生成方案/提案

**原则**：
- 优先引用 🔵 智讯自有案例
- 如需引用 🔴⚪ 内容，明确标注"参考思路，非直接引用"
- 每个核心论点必须有 `[R: 文件名]` 标注

**步骤**：
1. 理解客户需求
2. 搜索相关案例和方法论
3. 提取可复用的策略框架
4. 结合客户具体情况适配
5. 输出带引用的方案

### 5.2 生成白皮书/文章

**步骤**：
1. 确定主题
2. 搜索相关页面（案例+方法论）
3. 构建叙事结构
4. 填充内容（带引用）
5. 添加智讯观点作为"洞察层"

- 查看更新的页面
- 浏览知识网络
- 手动调整格式

## 8. 常用命令

### 搜索
```
# 搜索案例
grep -r "关键词" cases/ --include="*.md"

# 搜索方法论
grep -r "关键词" methodology/ --include="*.md"
```

### 统计
```
# 统计案例数量
ls cases/*.md | wc -l

# 统计方法论数量
ls methodology/*.md | wc -l
```
