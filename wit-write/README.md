# WIT 写作助手

## 这是什么

一个帮你写案例文章和方法论文章的 AI 写作助手。不是模板填充器——它会独立思考文章应该怎么写。

## 怎么用

在 Claude Code 中输入 `/wit-write` 触发。

## 工作流程

```
你丢材料进来 → AI 提取事实 → AI 追问补充 → AI 给你文章蓝图 → 你确认 → AI 写全文
```

具体来说：

1. **你提供材料** — 任何形式：聊天记录、PPT 笔记、项目文档、旧文章、甚至几句话都行，不需要填表格
2. **AI 提取事实** — 从你的材料中提取项目背景、决策过程、效果数据等，告诉你哪些信息够了、哪些缺了
3. **AI 引导补充** — 针对缺失的信息问你，问的是故事不是字段（比如「当时还考虑过什么方案？为什么没选？」）
4. **AI 给文章蓝图** — 不是直接写全文，而是先给你看文章结构和叙事逻辑，每节写清楚「这段要达到什么效果」
5. **你确认方向** — 检查结构和叙事线是否准确，提出调整
6. **AI 写全文** — 基于确认的蓝图写作，附带待确认事项清单
7. **回写知识库**（可选）— 确认终稿后自动保存到飞书云盘 wiki

## 你需要准备什么

最少只需要：**一段关于项目的描述**（哪怕是口头的几句话）。

如果你能提供以下任何一项，文章质量会更高：
- 项目背景和目标
- 策略选择的过程（考虑过什么、排除了什么、为什么选这条路）
- 执行中的关键决策和踩坑经验
- **效果数据**（必须有数字，没有数字的效果不算效果）

## 输出格式

每次输出包含：
- **文章正文** — markdown 格式，约 70% 完成度的初稿
- **YAML frontmatter** — 建议的元数据
- **待确认清单** — 标有 `❓` 的需要你补充确认的事项
- **交叉引用建议** — 哪些知识库页面应该和本文互相链接

## 写作原则

- **事实唯一**：所有内容必须有来源，编不出来。没有素材的地方会用 `[⚠️ 需补充]` 标记
- **决策链 > 结论**：展示「为什么选这条路」，不只是「做了什么」
- **像聊天不像报告**：用同行对话的语言，不是公文
- **数据说话**：效果必须有数字，没有数字会标注待补充

## 和 wit-wiki 的关系

- `/wit-wiki` 负责「想清楚」— 基于知识库回答品牌策略问题
- `/wit-write` 负责「写出来」— 基于你的材料写案例文章

写作时 wiki 作为素材补充来源，但文章结构和判断完全基于你的材料独立做出。

## 知识库说明

写好的文章可以回写到飞书云盘的 wiki 中，形成知识飞轮：
```
wiki 供给素材 → 帮你写案例 → 新案例回写 wiki → wiki 更大 → 下次写作素材更丰富
```

回写需要飞书云盘写入权限。当前通过 lark-cli 接入飞书云盘。
