# WIT 案例素材采集表

> **用途**：填写后交给 `/wit-write`，Agent 会基于此生成案例初稿
> **原则**：写你记得的，不确定的留空，Agent 会用 `[⚠️ 待补充]` 标记

---

## 快速版（5 分钟）

只需要填写这部分，Agent 就能出一个 60 分的初稿。

### 基本信息
```
项目名称：
客户名称：
行业：
项目时间：YYYY-MM ~ YYYY-MM
项目负责人：
```

### 一句话说清楚
```
这个项目做了什么？（一句话，不超过 30 字）

例：为万科杭州做了一次城市级品牌战役「与未来一起杭州」
```

### 核心素材
```
请提供以下任意素材（有哪个给哪个）：
□ 项目提案 / 结项报告（PPT 或 PDF）
□ 项目群聊天记录截图
□ 数据报告 / 效果复盘
□ 项目相关的设计稿、视频、海报
□ 其他你觉得重要的材料

素材路径或直接粘贴在这里：
```

## 完整版（15 分钟）

填写这部分，Agent 能出一个 85 分的初稿。**越详细，初稿质量越高。**

### 一、项目背景

```
1. 客户当时的处境是什么？
   （市场环境、竞争格局、客户面临的具体问题）


2. 客户为什么找我们？
   （是竞标赢的？老客户续约？客户主动找上门？）


3. 项目的核心挑战是什么？
   （用一句话描述最难的那个点）

```

### 二、洞察发现（最关键的部分）

```
1. 你们在项目中发现了什么反直觉的东西？
   （就是那种「咦，原来不是我们想的那样」的时刻）


2. 这个洞察是怎么发现的？
   □ 数据分析  → 什么数据？
   □ 用户调研  → 调研了谁？发现了什么？
   □ 案例对比  → 对比了什么案例？
   □ 团队讨论  → 讨论中谁说了什么触发了这个洞察？
   □ 客户反馈  → 客户说了什么让你重新思考？
   □ 灵光一现  → 当时的场景是什么？

3. 这个洞察如何改变了策略方向？
   （之前打算怎么做？洞察之后改成了什么？）

```

### 三、策略决策（第二关键的部分）

```
1. 当时摆在桌面上有几个方案？
   方案A：
   方案B：
   方案C：（如果有）

2. 最终选了哪个？为什么？
   （排除其他方案的具体原因）


3. 用了公司的哪个方法论？
   □ 反碎片  □ 品牌定位  □ 消费者洞察  □ 其他：
   在哪个环节用了？怎么用的？

```

### 四、执行过程

```
1. 项目分几个阶段？每个阶段做了什么？
   阶段1（时间：    ）：
   阶段2（时间：    ）：
   阶段3（时间：    ）：

2. 执行中最关键的一个动作是什么？
   （如果没有这个动作，项目就不会成功）


3. 有没有什么意外情况？
   （计划赶不上变化的时刻）

```

### 五、效果数据（必须有数字）

```
请提供具体数据（有的填，没有的留空）：

传播数据：
- 曝光量/阅读量：
- 互动量（点赞/评论/转发）：
- 媒体报道数量：
- 爆款内容（如有 10w+ 等）：

业务数据：
- 销售/转化数据：
- 客户满意度/反馈：

品牌数据：
- 品牌认知度变化：
- 行业奖项/认可：

其他你觉得重要的数据：
```

### 六、踩坑与复盘（最有价值的部分）

```
1. 项目中踩了什么坑？
   （做错了什么？什么没按计划走？）


2. 哪些假设被证伪了？
   （之前以为会怎样，结果不是）


3. 如果重来一次，你会做什么不同的决策？
   （这是其他项目最能复用的部分）


4. 客户最后怎么说？
   （客户的真实反馈，不是你包装过的）

```

### 七、关联信息

```
1. 这个项目和公司哪个方法论最相关？
   □ 反碎片  □ 品牌定位  □ 消费者洞察  □ 其他：

2. 和公司哪个历史项目最像？
   项目名称：

3. 竞品中有没有值得参考的类似案例？
   竞品名称：

4. 这个项目适合放在哪个行业分类？
   □ 房地产  □ 快消  □ 金融  □ 科技  □ 医疗  □ 其他：
```

## 使用说明

### 怎么提交
1. 复制本文件，填写后保存为 `素材-{项目名称}.md`
2. 在 Claude Code 中输入 `/wit-write`
3. 把填写好的文件路径告诉 Agent
4. Agent 会基于你的素材 + 知识库生成初稿

### 填写原则
- **写你记得的**：不确定的留空，不要猜
- **写真实的**：不要包装，踩坑经验比成功经验更有价值
- **写具体的**：「微博话题 6000 万」比「传播效果显著」有用 100 倍
- **写过程的**：「怎么发现的」比「发现了什么」重要

### 快速版 vs 完整版
- **快速版**：适合日常写稿，5 分钟搞定，Agent 出 60 分初稿
- **完整版**：适合重要案例、公众号发布、客户提案，15 分钟，Agent 出 85 分初稿
- **两者都填**：Agent 会优先使用完整版信息

### 信息缺口处理
Agent 生成初稿时，会用以下标记提示你补充：
- `[⚠️ 待补充数据]` — 需要具体数字
- `[⚠️ 待补充：xxx]` — 需要具体信息
- `❓` — 需要你确认的事实

你只需要关注这些标记，补充后就是终稿。
